# Thunderbird Pro — Daily Support Reports

Auto-generated by `.github/workflows/tbpro-daily.yml`, which runs `scripts/tbpro_daily.py --public` every day at 20:00 UTC (4pm EDT / 3pm EST) and commits the result here.

- `LATEST.md` — most recent day (always points to the newest report)
- `YYYY-MM-DD.md` — one file per day, dated by the report's ET calendar day
- Each report covers the 24h window ending at 4pm ET on the report date

## What's in each report

- Cumulative + 24-hour ticket volume and status breakdown
- CSAT for the last 24h and since launch (Zendesk satisfaction ratings)
- Theme breakdown of the last 24h tickets (login, signup, refunds, etc.)
- Negative CSAT detail (reason + comment), PII-redacted
- Refund / cancellation tickets with redacted reasons
- New FeatureOS ideas in the last 24h + since launch

## PII handling

The `--public` flag scrubs emails, phone numbers, IPs, and common name patterns (greetings and signoffs) from all user-supplied text before committing. Ticket IDs, themes, counts, FeatureOS titles, and CSAT reason strings are not redacted — staff need them to act on the report.

## To run on demand

From the repo's Actions tab → "TB Pro Daily Report" → "Run workflow", or locally:

```sh
python3 scripts/tbpro_daily.py --public --out-dir reports/tbpro
```

Local runs need `~/.config/zendesk/credentials` (with `email=`, `token=`, `subdomain=tbpro`) and `~/.featureos.yaml`. CI uses repo secrets.
